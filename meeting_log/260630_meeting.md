## 🗣️ 주요 논의 내용

### 1. 징글 생성 모델 점검

- 음악을 별도로 학습한 뒤 보컬 생성 모델을 활용하여 믹싱했으나, 결과가 기대에 미치지 못했다.

- 모델을 한 번에 결합하기보다 각 모델의 성능을 개별적으로 점검할 필요가 있다.

- 목표로 하는 입력과 출력을 다음과 같이 명확히 정의하였다.
  - **Input:** 프롬프트
  - **Output:** 생성된 오디오 파일(MP3)

- 사용한 모델이 실제로 파인튜닝에 활용된 사례가 있는지 확인할 필요가 있다.

- 기존 활용 사례에서 사용한 데이터 규모, 생성 결과, 사용자 평가 등을 함께 확인해야 한다.

- 현재 사용한 데이터 74개는 학습에 부족할 가능성이 크며, 기존 모델의 성능까지 저하시킬 수 있다는 의견이 제시되었다.

- 에포크 수를 늘렸을 때 결과가 개선되는지 실험할 필요가 있다.

- 이를 통해 모델 자체의 한계인지, 데이터 부족으로 인한 문제인지 구분해야 한다.

- 음의 높낮이, 톤, 장조와 단조 등을 변형하여 데이터를 증강하는 방법은 직접 실험을 통해 효과를 확인해야 한다.

- 보컬 믹싱은 멜로디 생성 결과가 먼저 확보되어야 진행할 수 있다.

- 멜로디 생성용 모델에 보컬까지 입히는 것은 모델 구조를 직접 수정하지 않는 이상 어려울 가능성이 있다.

- 학습 성능을 개선하려면 에포크 수를 늘리거나 데이터 규모를 확대하는 방안을 고려해야 한다.

- 가사까지 생성할 수 있는 음악 생성 모델의 존재 여부와 생성 가능한 오디오 길이도 확인할 필요가 있다.

- 모델이 제공하는 자체 샘플과 데모 버전의 성능을 먼저 확인해야 한다.

- 직접 수집한 멜로디 데이터와 가사가 포함된 MP3 데이터를 학습했을 때의 결과를 비교할 필요가 있다.

- 동요 데이터를 징글 데이터와 함께 사용한 것이 학습 결과에 영향을 주었을 가능성도 검토하였다.

- 프로젝트 기간을 고려하여 멜로디는 직접 생성하고, 보컬은 상용화된 AI 모델을 활용하는 방안도 논의하였다.

- 기존 사례에서도 가사를 직접 적용하기 어려워 허밍 방식으로 전환한 경우가 있었다.

### 2. 공사장 안전 관리 프로젝트

- 공사 현장에서 안전 장비의 사용 여부를 확인하여 근로자의 안전을 확보하는 것을 프로젝트의 목적으로 설정하였다.

- 사고 발생 시 관리자의 책임이 크다는 점을 고려하여, 주요 사용자를 현장 관리자로 설정하였다.

- 관리자의 관점에서 현장 안전 상태를 쉽게 확인할 수 있는 시스템으로 접근하기로 하였다.

- 주요 탐지 대상은 다음 두 가지로 설정하였다.
  1. 안전고리
  1. 안전난간

- 안전모와 안전조끼 탐지는 비교적 흔한 주제이므로 우선순위에서 제외하였다.

- 안전고리는 크기가 작아 이미지에서 인식되지 않는 경우가 많지만, 미체결로 인한 사고 사례가 많아 가장 중요한 탐지 대상으로 판단하였다.

- 우선 안전고리 체결 여부를 탐지하고, 이후 안전난간 설치 여부를 탐지하는 방향으로 진행하기로 하였다.

- 안전난간이 설치되지 않은 현장이 있을 수 있으므로, 실제 설치 여부와 규정에 맞게 설치되었는지를 확인하는 방안을 논의하였다.

- 안전난간 사이의 거리 측정 등 구체적인 규정 확인 기능도 검토 대상으로 언급되었다.

- 데이터는 외국과 국내 데이터를 함께 수집하고, 촬영 거리가 가까운 이미지와 먼 이미지를 모두 포함하기로 하였다.

- 안전고리가 체결된 이미지를 중심으로 수집하며, 철골 구조물이 아닌 비계 작업 상황으로 범위를 제한하기로 하였다.

- 데이터 수집 경로로 구글 이미지 검색과 유튜브 영상을 활용하기로 하였다.

- 라벨링은 AI-Hub 데이터의 라벨링 기준을 확인한 뒤 가능한 한 동일한 기준으로 진행할 필요가 있다.

- 다양한 YOLO 모델을 적용해 성능을 비교하기로 하였다.

- Roboflow에서 데이터 증강 시 원본 데이터의 약 3배까지 확대할 수 있으므로, 최종 3,000~4,000장의 데이터를 확보하기 위해 원본 이미지 약 1,000장을 수집하는 방향을 논의하였다.

- 안전고리가 비계에 연결된 부분만 탐지 대상으로 볼 것인지 추가 검토가 필요하다.

## 🎯 회의 결과

### 징글 생성 모델

- 현재 생성 결과가 좋지 않은 원인을 확인하기 위해 모델별 성능을 개별적으로 점검하기로 하였다.

- 모델 자체의 한계와 데이터 부족의 영향을 구분하기 위해 에포크 수와 데이터 규모를 조정하여 실험하기로 하였다.

- 음의 높낮이, 톤, 장조와 단조 변형을 활용한 데이터 증강도 실험 대상으로 검토하기로 하였다.

- 모델을 선정할 때는 파인튜닝 사례, 자체 샘플 및 데모 결과를 우선 확인하기로 하였다.

- 프로젝트 기간을 고려하여 멜로디 생성과 보컬 생성을 분리하고, 보컬 부분에 상용 AI를 활용하는 방안도 고려하기로 하였다.

### 공사장 안전 관리 프로젝트

- 현장 관리자를 주요 사용자로 설정하고, 공사 현장의 안전 장비 사용 여부를 확인하는 프로젝트로 추진하기로 하였다.

- 탐지 우선순위는 안전고리 체결 여부를 1순위, 안전난간 설치 여부를 2순위로 설정하였다.

- 비계 작업 현장의 이미지를 중심으로 국내외 데이터를 수집하기로 하였다.

- 원본 이미지 약 1,000장을 수집하고, 데이터 증강을 통해 3,000~4,000장 규모의 데이터셋을 구축하는 방향으로 진행하기로 하였다.

- AI-Hub 라벨링 기준을 확인한 뒤 라벨링을 진행하고, 여러 YOLO 모델을 적용하여 성능을 비교하기로 하였다.
