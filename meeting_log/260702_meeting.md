## 🗣️ 주요 논의 내용

### 1. 세그멘테이션 라벨링 기준

- `hook` 클래스의 범위를 안전고리와 랜야드를 포함하는 형태로 정의하는 방안을 논의하였다.

- 객체가 연결된 상태는 세그멘테이션 마스크의 픽셀이 서로 겹치는지를 기준으로 판단하는 방안을 검토하였다.

- 안전고리와 랜야드를 하나의 `hook` 클래스로 설정할지, `hook`과 `line`으로 분리할지도 함께 검토하였다.

### 2. 작업자 탐지 기준

- 기존 모델이 작업자를 정확하게 탐지하지 못하는 문제가 있어, `worker` 대신 `person` 클래스를 활용하는 방안을 검토하기로 하였다.

- 우선 작업자의 전신이 포함된 이미지만 사용하여 학습한 뒤 탐지 성능을 확인하기로 하였다.

- 이후 `person` 클래스만 사용한 경우와 안전모 등의 클래스를 추가한 경우를 비교하는 방향을 논의하였다.

### 3. 소형 객체 및 라인 탐지

- 안전고리와 같이 크기가 작은 객체를 탐지할 수 있는 모델로 RT-DETR을 검토하였다.

- 랜야드와 같은 선 형태의 객체를 탐지하기 위해 라인 검출과 에지 검출 기법을 적용하는 방안도 논의하였다.

- 단일 모델만 사용하는 방식뿐 아니라 여러 모델의 결과를 종합하거나 기존 모델을 활용하는 방안도 검토하였다.

### 4. 모델 학습 및 검증 방향

- 정면에서 촬영된 이미지와 같이 비교적 탐지가 쉬운 데이터부터 학습과 검증을 진행하기로 하였다.

- 다양한 모델과 라벨링 방식을 적용하여 결과를 비교한 뒤 구체적인 방향을 설정하기로 하였다.

- 다른 사람이 구축한 학습 데이터나 기존 데이터셋을 활용하는 방안도 검토하기로 하였다.

## 🎯 회의 결과

- 안전고리 세그멘테이션을 위해 `hook` 클래스를 안전고리와 랜야드를 포함하는 범위로 정의하는 방안을 우선 검토하기로 하였다.

- 객체 간 연결 여부는 세그멘테이션 마스크의 픽셀 중첩을 활용하여 판단하는 방향을 검토하기로 하였다.

- 작업자 탐지에는 `worker` 클래스 대신 `person` 클래스를 적용할 수 있는지 확인하기로 하였다.

- 전신이 명확하게 보이는 이미지와 정면 이미지 등 비교적 단순한 데이터부터 학습을 진행하기로 하였다.

- 안전고리와 같은 소형 객체 탐지를 위해 RT-DETR 등의 모델을 검토하고, 랜야드 탐지에는 라인 및 에지 검출 기법을 함께 고려하기로 하였다.

- 여러 모델과 기존 학습 데이터를 다양하게 적용한 뒤 결과를 비교하여 프로젝트 방향을 구체화하기로 하였다.
