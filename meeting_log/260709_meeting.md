## 🗣️ 주요 논의 내용

### 1. 작업자 안전 상태 판단 기준

- 안전고리가 체결되어 있지 않은 경우 작업자를 안전하지 않은 상태로 판단하기로 하였다.

- 하네스와 랜야드는 작업 중 의무적으로 착용해야 하는 장비이므로, 이를 안전 상태 판단에 어떻게 반영할지 논의하였다.

- 처음에는 안전고리 체결 여부만으로 작업자의 안전 여부를 판단하고, 하네스와 랜야드는 모델이 다양한 객체를 탐지할 수 있다는 점을 보여주는 용도로 활용하는 방안을 검토하였다.

- 하네스와 랜야드의 체결 여부를 안전 판단 기준에 포함하여 차별화하려 했으나, 랜야드가 주로 하네스의 등 부분에 연결되어 있어 작업자의 앞모습과 뒷모습에 따라 체결 여부를 확인하기 어렵다는 한계가 있었다.

- 해당 한계는 향후 고도화 과제로 제시하는 방향을 검토하였다.

- 하네스를 불편하다는 이유로 벗는 작업자가 있을 수 있으므로, 하네스의 존재 여부를 작업자별 안전 상태 판단에 포함하는 방안을 수용하였다.

- 최종적으로 안전고리 체결 여부와 하네스 착용 여부를 작업자의 안전 상태 판단에 활용하고, 랜야드의 활용 방식은 추가 검토가 필요한 과제로 정리하였다.

### 2. 하네스와 랜야드의 활용 방향

- 하네스는 현재 모델이 비교적 잘 탐지하고 있어 작업자별 안전 상태 판단에 활용하는 것이 적절하다고 보았다.

- 랜야드는 작업자가 멀어질 때 팽팽해져 위험할 가능성이 있으므로, 이를 별도로 구분하여 위험 신호에 활용하는 방안을 논의하였다.

- 다만 랜야드의 상태와 체결 여부를 실제로 어떤 기준으로 판별할지는 명확하지 않아 추가 검토가 필요하다.

### 3. 프로젝트 차별화 방향

- 현재 구현 가능한 기능과 구현이 어려운 기능을 구분한 뒤, 프로젝트의 차별점을 명확하게 설정해야 한다는 피드백을 받았다.

- 기존 논문 및 제품과 비교하여 프로젝트의 목적과 차별성을 구체적으로 정의할 필요가 있다.

- 광범위한 공사 현장과 저해상도 CCTV 영상에서도 안전고리 체결 여부를 탐지할 수 있다는 방향으로 프로젝트의 스토리를 구성하는 방안을 논의하였다.

- AI-Hub의 CCTV 이미지 데이터와 유튜브의 유사 CCTV 영상을 활용하여 다양한 작업 환경에서 모델을 검증하는 방안을 검토하였다.

- 비계 작업뿐 아니라 개구부 작업이나 학습에 포함되지 않은 구도와 배경 등 다양한 공사 현장에 적용할 수 있는 모델로 확장하는 것도 차별화 방향으로 제시되었다.

### 4. 분류 모델 고도화

- 안전고리 체결 여부를 판단하는 분류 모델 자체를 고도화하는 것이 중요한 과제로 정리되었다.

- 하나의 분류 모델만 사용하는 방식 외에도 앙상블 모델을 적용하는 방안을 검토하였다.

- ViT, EfficientNet 등 여러 모델을 적용하고 정량적 성능을 비교하여 더 적합한 모델을 선택하는 방안이 제안되었다.

- 모델을 그대로 사용하는 데 그치지 않고, 기존 모델의 레이어나 Feature Map을 목적에 맞게 활용하는 방안을 검토해야 한다는 피드백을 받았다.

- ViT의 마지막 Feature Map을 추출한 뒤 이를 분류에 활용하는 방안도 논의하였다.

- 기존 레이어를 변경하거나 새로운 파라미터를 추가할 경우 초기화된 파라미터가 생성될 수 있으므로, 충분한 데이터가 필요하다는 의견이 제시되었다.

- 데이터가 부족한 경우 소규모 데이터를 단계적으로 적용하면서 어떤 방식이 더 효율적인지 비교해야 한다.

- 기존 ViT 실험은 데이터가 너무 적었고 모델의 크기가 컸기 때문에, 전체 모델을 학습하기보다 출력 분류 부분에 집중했어야 한다는 문제가 제기되었다.

### 5. 객체 탐지 모델 고도화

- RF-DETR은 현재 객체 탐지에 사용하고 있으며, 탐지 결과를 분류 모델의 입력으로 활용하고 있다.

- 단순히 모델의 최종 출력만 사용하는 것에서 나아가, RF-DETR 내부의 Feature Map을 확인하고 프로젝트 목적에 맞게 활용하는 방안을 검토하였다.

- RF-DETR 또는 다른 소형 객체 탐지 모델의 중간 특징을 활용하여 분류 성능을 높일 수 있는지도 확인할 필요가 있다.

- ViT를 비롯해 작은 객체를 잘 탐지할 수 있는 모델을 추가로 실험하는 방안도 제안되었다.

- 객체 탐지를 개선하려면 CCTV 이미지 데이터를 추가로 학습시키는 방안을 검토하기로 하였다.

### 6. 선행연구 및 베이스라인

- 모델의 성능을 판단하려면 비교 기준이 되는 베이스라인이 필요하다는 피드백을 받았다.

- 기존 논문과 제품을 확인하고 정량적 성능 차이를 비교해야 한다.

- 선행연구에서 사용한 평가 지표를 확인한 뒤 현재 모델의 성능이 적절한 수준인지 판단할 필요가 있다.

- 단순히 성능 수치만 제시하기보다, 기존 연구와 비교했을 때 프로젝트가 어떤 점에서 다른지 명확하게 설명해야 한다.

### 7. 오탐 및 미탐에 대한 판단 기준

- 안전하지 않은 상황을 안전하다고 판단하면 대형 사고로 이어질 수 있으므로, 모델의 판단 기준을 보수적으로 설정할 필요가 있다.

- 실제로는 안전하지만 위험하다고 알림을 보내는 오탐은 작업자에게 불편을 줄 수 있으나, 위험한 상황을 안전하다고 판단하는 오류보다는 감수할 수 있다는 의견이 제시되었다.

- 암 진단이나 스팸 탐지 사례와 같이, 안전 문제에서는 위험 상황을 놓치지 않는 방향으로 판단 기준을 설정하는 것이 필요하다.

### 8. 안전고리 체결 분류의 주요 문제점

- 안전고리가 실제로 체결되지 않고 작업자의 허리에 있는 상태에서도 안전바와 겹쳐 보이면 `connected`로 분류되는 문제가 확인되었다.

- AI-Hub의 저화질·원거리 이미지뿐 아니라 유튜브의 고화질·근거리 영상에서도 동일한 문제가 발생하였다.

- 이를 통해 화질이나 촬영 거리보다 안전고리의 배경에 무엇이 위치하는지가 분류 결과에 영향을 주는 것으로 판단하였다.

- 안전고리의 배경이 안전바이면 `connected`, 하네스나 다른 객체이면 `unconnected`로 분류하는 경향이 확인되었다.

- 안전고리와 안전바의 단순한 중첩이나 배경을 기준으로 판단하지 않고, 안전고리가 선의 앞에 있는지 뒤에 있는지를 분류하는 방향을 검토하였다.

- 우선 영상에서 일정 시간 동안 상태가 유지되는지를 기준으로 연결 여부를 판단하는 방안을 고려하였다.

- 미체결 상태가 장시간 지속되는 경우도 있을 수 있으므로, 해당 문제는 모델의 한계로 제시할지 추가 검토가 필요하다.

- 안전바에 걸려 있는 안전고리와 유사한 객체를 `hook`으로 잘못 탐지하고, 이후 `connected`로 분류하는 문제가 확인되었다.

- 이는 체결 여부 분류 이전에 잘못된 객체를 안전고리로 탐지한 문제로, 객체 탐지 모델의 개선이 필요하다.

- 작업자가 너무 멀리 있는 경우 안전고리를 탐지하지 못하는 문제가 확인되었다.

- 이 문제는 현재 모델뿐 아니라 대부분의 모델에서 발생할 가능성이 있는 한계로 검토되었다.

- 기존 연구에서는 안전고리의 닫힘 여부도 분류한 사례가 있었다.

- 안전고리가 완전히 닫히지 않은 애매한 상태는 단일 이미지보다 영상을 활용하면 판단 가능성이 높아질 수 있다는 의견이 제시되었다.

## 🎯 회의 결과

- 작업자의 안전 상태는 안전고리 체결 여부와 하네스 착용 여부를 중심으로 판단하는 방향으로 정리하였다.

- 랜야드와 하네스의 체결 여부는 시야와 구도에 따른 한계가 있어 향후 고도화 과제로 제시하기로 하였다.

- 안전 분야의 특성을 고려하여 위험 상황을 놓치지 않는 보수적인 판단 기준을 적용하기로 하였다.

- 분류 문제와 객체 탐지 문제를 구분하여 각각 개선하기로 하였다.
  - 안전고리가 안전바와 겹쳐 `connected`로 분류되는 문제는 분류 모델의 문제로 정의하였다.
  - 안전고리와 유사한 객체를 `hook`으로 탐지하는 문제는 객체 탐지 모델의 문제로 정의하였다.

- 안전고리 체결 분류는 ViT, EfficientNet 및 앙상블 모델을 적용하여 성능을 비교하기로 하였다.

- RF-DETR과 분류 모델의 Feature Map을 활용하여 목적에 맞는 출력 구조를 구성하는 방안을 검토하기로 하였다.

- 객체 탐지 성능 향상을 위해 CCTV 이미지 데이터를 추가로 학습하는 방안을 검토하기로 하였다.

- AI-Hub CCTV 이미지와 유튜브 유사 영상을 활용하여 저해상도 및 다양한 작업 환경에서 모델을 검증하기로 하였다.

- 기존 논문 및 제품을 베이스라인으로 설정하고 정량적 성능과 차별점을 비교하기로 하였다.

- 광범위한 공사 현장과 저해상도 CCTV 환경에서도 적용 가능한 안전고리 체결 여부 탐지 모델이라는 방향으로 프로젝트를 구성하기로 하였다.
