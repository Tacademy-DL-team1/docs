# 🧠 안전고리 및 랜야드 라벨링 방식 개선 회의록

> **💡 Summary**
  안전고리와 랜야드를 하나의 클래스로 라벨링한 결과 안전고리만 있는 이미지의 탐지 성능이 낮아진 원인을 검토하고, 두 객체를 분리하여 라벨링하는 방향을 논의하였다.

- **회의 날짜:** 2026-07-06

## 📌 회의 안건

- 안전고리와 랜야드를 함께 라벨링한 방식의 문제점 확인

- 생명줄과 안전고리의 라벨링 방식 검토

## 🗣️ 주요 논의 내용

### 1. 기존 라벨링 방식

- 기존에는 안전고리의 고리 부분과 랜야드를 하나의 `hook` 클래스로 묶어 라벨링하였다.

- 생명줄과 안전고리를 한 번에 라벨링하는 방식과 랜야드와 안전고리를 구분하는 방안을 검토하였다.

### 2. 기존 방식의 문제점

- 고리 부분과 랜야드를 하나의 클래스로 학습한 결과, 안전고리만 보이는 이미지에서는 모델이 객체를 제대로 탐지하지 못했다.

- 학습 데이터에 안전고리만 있는 이미지가 많지 않았던 점이 탐지 성능에 영향을 준 것으로 판단하였다.

- 랜야드는 종류가 다양하고 작업자의 몸 주변에 복잡하게 배치된 경우가 많아, 하나의 클래스로 학습하기 어려웠을 가능성이 있다.

### 3. 라벨링 방식 개선 방향

- 안전고리와 랜야드는 형태와 데이터 특성이 다르므로 별도의 클래스로 구분하여 라벨링하는 것이 적절할 수 있다는 의견을 도출하였다.

- 기존 통합 라벨링 실험을 통해 안전고리와 랜야드를 분리해야 할 필요성을 확인하였다.

## 🎯 회의 결과

- 안전고리의 고리 부분과 랜야드를 하나의 `hook` 클래스로 라벨링한 방식은 안전고리 단독 탐지에 적합하지 않을 수 있음을 확인하였다.

- 안전고리만 포함된 학습 데이터가 부족하고 랜야드의 형태가 다양했던 점이 탐지 성능 저하의 원인으로 검토되었다.

- 향후 안전고리와 랜야드를 서로 다른 클래스로 구분하여 학습하는 방향을 검토하기로 하였다.

- 기존 실험을 통해 라벨링 기준을 개선할 수 있는 방향을 확인하였다.
