# 🧠 안전고리 체결 판별 모델 최종 선정 및 발표 구성 회의록

> **💡 Summary**
  ViT와 EfficientNet의 체결 여부 분류 성능과 처리 속도를 비교하여 최종 모델을 선정하는 기준을 논의하였다. 모델별 정량 지표와 실제 예시를 함께 제시하고, 객체 탐지부터 크롭·분류까지 전체 처리 속도와 영상 적용 결과를 최종 발표에 포함하기로 하였다.

- **회의 안건:** ViT·EfficientNet 비교, 모델 처리 속도 측정 및 최종 발표 자료 구성

## 🗣️ 주요 논의 내용

### 1. 현재 데이터 및 모델 진행 상황

- 미체결 데이터를 추가로 수집한 뒤 증강하여 ViT 기본 모델을 학습한 결과, 예상보다 좋은 성능을 확인하였다.

- 체결 데이터는 약 1,500장, 미체결 데이터는 약 460장이었으며, 미체결 데이터를 증강하여 두 클래스의 비율을 약 1:1로 맞추었다.

- EfficientNet은 모델 구조를 수정하여 학습을 진행하였다.

- 기본 성능 지표를 비교했을 때 ViT가 EfficientNet보다 높은 결과를 보여, 최종 모델 선정 기준에 대한 논의가 필요해졌다.

### 2. 최종 모델 선정 기준

- 안전성을 우선할 경우 분류 성능이 더 좋은 ViT를 선택하는 논리를 검토하였다.

- 향후 CCTV 환경에 적용하는 것을 고려하면 상대적으로 가볍고 빠른 EfficientNet을 선택하는 논리도 가능하다고 보았다.

- 단순히 가장 높은 성능을 기록한 모델을 선택하기보다, 활용 목적에 따라 성능과 처리 속도를 함께 고려해야 한다는 의견이 제시되었다.

- 여러 실험 결과를 비교한 뒤 적정 수준의 성능과 속도를 만족하는 모델을 선택할 필요가 있다.

- 공사 현장은 비교적 고성능 하드웨어를 사용할 수 있는 환경이므로, 하드웨어 성능을 높여 모델의 처리 속도를 보완하는 방안도 고려할 수 있다.

- ViT는 기본 모델을 그대로 사용한 것으로 보였으나, 실제 코드에서는 이미 일부 수정이 적용되었을 가능성이 있어 다시 확인하기로 하였다.

- 최종 선택 과정에서 ViT와 EfficientNet의 비교 결과를 시각적으로 제시할 필요가 있다.

### 3. 하드웨어별 처리 속도 비교

- ViT와 EfficientNet의 성능뿐 아니라 하드웨어별 처리 속도를 비교하기로 하였다.

- 비교 대상 하드웨어는 다음과 같다.
  - T4
  - L4
  - A100

- 모델별로 이미지 한 장을 처리하는 시간을 측정하기로 하였다.

- 영상 적용 가능성을 확인하기 위해 초당 처리할 수 있는 최대 프레임 수를 계산해야 한다.

- 60FPS 영상은 프레임당 약 `1/60초`, 30FPS 영상은 프레임당 약 `1/30초` 안에 처리가 가능한지 확인할 필요가 있다.

- 처리 속도가 부족한 경우 다음 방안을 검토하기로 하였다.
  - 더 높은 성능의 하드웨어 활용
  - 입력 영상의 프레임 수 조정
  - 성능과 속도의 균형을 고려한 모델 선택

### 4. 단계별 처리 속도 측정

- 전체 시스템의 실시간 적용 가능성을 확인하기 위해 처리 단계를 구분하여 속도를 측정하기로 하였다.

- 측정 대상은 다음과 같다.
  1. 객체 탐지만 수행했을 때의 처리 속도
  1. 체결 여부 분류만 수행했을 때의 처리 속도
  1. 객체 탐지 후 크롭하고 분류까지 수행했을 때의 전체 처리 속도

- 객체 탐지 모델과 분류 모델 모두 성능 지표와 함께 처리 시간을 제시해야 한다.

- 영상 입력부터 객체 탐지, 크롭, 체결 여부 분류까지 소요되는 전체 시간을 확인하기로 하였다.

### 5. 모델 비교 결과 제시 방식

- 정량적인 평가 지표만 제시하지 않고 실제 이미지 사례를 함께 보여줄 필요가 있다.

- 두 모델이 모두 잘 분류한 사례, 한 모델만 잘 분류한 사례, 두 모델 모두 잘못 분류한 사례를 직접 비교하기로 하였다.

- 모델별 결과는 수치와 실제 이미지를 함께 제시하여 차이를 명확하게 보여주는 방향으로 구성하기로 하였다.

- 앞에서 정의한 탐지 대상과 분류 기준을 바탕으로 각 모델이 해당 객체와 상태를 정확히 판단했는지 확인할 수 있도록 구성해야 한다.

- 분류 모델 발표 자료에는 다음 내용을 포함하기로 하였다.
  - 분류 성능 지표
  - 처리 속도
  - 정확하게 분류한 이미지
  - 잘못 분류한 이미지
  - 모델별 비교 결과

### 6. 평가 지표 설명

- 발표 자료의 평가 지표 부분에는 각 지표의 의미를 간단한 사례와 함께 설명하는 방안을 검토하였다.

- 최종 모델을 선택한 이유가 성능 지표와 처리 속도를 통해 자연스럽게 드러나도록 구성해야 한다.

- 객체 탐지 모델과 분류 모델의 지표를 각각 제시하고, 최종 시스템 성능과의 관계를 설명할 필요가 있다.

### 7. 발표 자료 구성 방향

- 발표 초반에는 프로젝트가 해결하려는 문제와 안전고리 체결 여부를 탐지해야 하는 이유가 명확하게 전달되어야 한다.

- 안전고리 체결 여부가 프로젝트의 핵심 차별점이라는 점을 중심으로 내용을 구성해야 한다.

- 선행연구를 검토한 결과 어떤 한계가 있었고, 이를 바탕으로 본 프로젝트에서 무엇을 구현하려 했는지 명확하게 설명해야 한다.

- 프로젝트의 탐지 대상, 관련 용어, 작업 상황과 라벨링 기준을 자연스럽게 연결해야 한다.

- 동일하거나 유사한 내용을 반복하여 제시하지 않도록 전체 발표 흐름을 점검할 필요가 있다.

- 먼저 실제 예시를 제시한 뒤 정량적 성능 지표와 최종 모델 선택 결과를 설명하는 구성도 검토하였다.

- 모델별 사례는 한 페이지 안에서 비교할 수 있도록 이미지 중심으로 구성하는 방안이 제안되었다.

### 8. 최종 영상 시연

- 최종 결과는 정지 이미지뿐 아니라 영상에 적용한 형태로 제시하기로 하였다.

- 영상에는 모델이 잘 판단하는 사례와 잘못 판단하는 사례를 모두 포함해야 한다.

- 영상 결과를 통해 현재 시스템이 수행할 수 있는 기능과 한계를 함께 보여주기로 하였다.

- 최종 영상에서는 다음 처리 과정을 확인할 수 있도록 구성하기로 하였다.
  1. 영상 입력
  1. 작업자 및 안전 장비 객체 탐지
  1. 안전고리 영역 크롭
  1. 체결 여부 분류
  1. 작업자 안전 상태 표시

- 영상 처리 시간과 객체 탐지부터 분류까지의 전체 처리 결과도 함께 제시하기로 하였다.

- 실제 시스템을 예상할 수 있도록 데모 형태로 구성하는 방향을 검토하였다.

## 🎯 회의 결과

- ViT와 EfficientNet의 최종 선택은 분류 성능뿐 아니라 모델 크기, 처리 속도와 실제 활용 목적을 함께 고려하여 결정하기로 하였다.

- 안전 판단 성능을 우선하면 ViT, 실시간 CCTV 적용과 경량성을 우선하면 EfficientNet을 선택할 수 있으며, 실험 결과를 바탕으로 적정 모델을 선정하기로 하였다.

- T4, L4, A100 환경에서 모델별 이미지 한 장의 처리 시간과 최대 처리 가능 FPS를 비교하기로 하였다.

- 객체 탐지, 분류, 객체 탐지·크롭·분류 전체 과정의 처리 속도를 각각 측정하기로 하였다.

- 모델 비교 시 성능 지표와 함께 잘 분류한 사례, 잘못 분류한 사례, 모델별 차이가 나타나는 사례를 이미지로 제시하기로 하였다.

- ViT 모델에 실제로 어떤 수정이 적용되었는지 코드를 다시 확인하기로 하였다.

- 최종 발표 자료는 프로젝트 목적, 선행연구의 한계, 탐지 대상과 라벨링 기준, 모델 성능 비교, 최종 모델 선정의 흐름이 자연스럽게 이어지도록 수정하기로 하였다.

- 최종 결과는 객체 탐지부터 체결 여부 분류까지 전체 과정을 확인할 수 있는 영상으로 제시하기로 하였다.

- 영상에는 모델이 잘 판단하는 상황과 잘못 판단하는 상황을 모두 포함하여 성능과 한계를 함께 보여주기로 하였다.
